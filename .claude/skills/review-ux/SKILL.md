# Review UX — UX レビュー

変更が利用者にとって使いやすいかを確認し、改善点を指摘する。

## 対象の特定

1. 変更ファイルを取得する:

```sh
git diff main --name-only
```

2. 変更ファイルの全体を読む（UI コードは diff だけでは全体像が掴めない）
3. 関連するコンポーネント・CSS を読み、画面上での見え方・動き方を把握する

## 確認観点

### 操作フィードバック

- リンクやボタンにホバー・アクティブ状態のスタイルがあるか
- クリック可能な要素が視覚的にクリック可能だとわかるか（カーソル、色、下線など）
- アンカーリンクでのスムーズスクロールが動作するか
- 外部リンクと内部リンクの区別がつくか（↗ アイコン等）

### レスポンシブ対応

- モバイル（768px以下）での表示が崩れていないか
- タッチターゲットが十分な大きさか（最低 44x44px 推奨）
- テキストがモバイルで読みやすいサイズか
- 横スクロールが発生していないか
- ナビゲーションがモバイルで使いやすいか

### キーボード操作

- フォーカス可能な要素に適切なフォーカススタイルがあるか
- フォーカス順序が視覚的な配置と一致しているか
- `a` タグや `button` が正しく使い分けられているか（ナビゲーションは `a`、アクションは `button`）

### 視覚的一貫性

- 既存セクションと見た目・余白・フォントサイズが揃っているか
- CSS 変数（`var(--color-*)`, `var(--font-*)`）を使って一貫した配色・書体になっているか
- ハードコードされた色やフォントがないか
- セクション番号・ドット・ラベルの体裁が他のセクションと統一されているか

### アクセシビリティ

- ナビゲーションやランドマークに `aria-label` があるか
- 色だけに頼らない情報伝達になっているか（色覚多様性への配慮）
- フォーカス時の視覚的インジケータ（アウトライン等）があるか
- 外部リンクに `rel="noopener noreferrer"` が付いているか
- セマンティック HTML（`header`, `nav`, `main`, `footer`, `section`）が適切に使われているか

## 出力

指摘事項を以下の形式で報告する:

```
### UX レビュー結果

**操作フィードバック**
- [ ] `ファイル名:行番号` — 指摘内容（改善案）

**レスポンシブ対応**
- [ ] ...

**キーボード操作**
- [ ] ...

**視覚的一貫性**
- [ ] ...

**アクセシビリティ**
- [ ] ...

該当カテゴリに指摘がなければそのカテゴリごと省略する。
全て指摘なしの場合: 「UX 上の問題は見つかりませんでした」
```

修正が求められている場合（issue-handler 経由）は、報告後に指摘箇所を修正する。
