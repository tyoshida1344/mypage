# Review Regression — デグレチェック

変更が既存機能を壊していないかを確認する。

## 対象の特定

1. 変更ファイルと差分を取得する:

```sh
git diff main --name-only
git diff main
```

2. 変更ファイルの全体を読む
3. 変更ファイルを参照・利用している関連ファイルを特定して読む。特に:
   - 変更したコンポーネントを利用しているページ・親コンポーネント
   - 変更した CSS 変数を使用している他のコンポーネント
   - 変更したデータ構造（`data/site.json`）を参照しているコンポーネント

## 確認観点

### コンポーネントの互換性

- props のシグネチャ（名前・型・デフォルト値）が変わっていないか
- slot の構造が変わり、親コンポーネント側の利用が壊れていないか
- emit されるイベントの名前・引数が変わっていないか
- `SectionWrapper` などの共通コンポーネントの挙動が維持されているか

### データ構造の互換性

- `data/site.json` のフィールド名・構造が変わっていないか（既存データが読めなくなる）
- `data/site.example.json` と `data/site.json` の構造が同期しているか
- `nuxt.config.ts` でのデータ読み込み処理が正しく動作するか
- `scripts/setup-data.sh` の挙動に影響がないか

### CSS の影響

- 変更・削除した CSS 変数（`assets/css/main.css`）が他のコンポーネントで使われていないか
- スコープ外に漏れるスタイル変更がないか（`:deep()` や グローバルスタイルの変更）
- レスポンシブブレークポイント（`768px`）の一貫性が保たれているか
- `var(--color-*)` の値変更が全体のカラースキームを壊していないか

### レイアウト・構造の影響

- `pages/index.vue` のコンポーネント配置順が意図通りか
- セクション間の border / padding の整合性が崩れていないか
- ヘッダーナビゲーションのリンク先（`#id`）と各セクションの `id` が一致しているか
- フッターのリンクが正しく動作するか

### ビルド・デプロイへの影響

- `nuxt.config.ts` の変更がビルドを壊さないか
- `netlify.toml` のビルドコマンドとの整合性が保たれているか
- 新しいモジュール追加時に `package.json` に正しく記載されているか

## 出力

指摘事項を以下の形式で報告する:

```
### デグレチェック結果

- [ ] `ファイル名:行番号` — 指摘内容（影響を受ける箇所: `関連ファイル:行番号`）

指摘なしの場合: 「デグレの懸念は見つかりませんでした」
```

修正が求められている場合（issue-handler 経由）は、報告後に指摘箇所を修正する。
